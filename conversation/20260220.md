  1. Passwordless SSH (SSH Key Auth)                                                                                                                            
                                                                                                                                                                
  Run this on your M1 Max (the machine you SSH from):                                                                                                           
                                                                                                                                                                
  # Generate an SSH key if you don't already have one                                                                                                           
  # Just press Enter for all prompts (no passphrase needed for convenience)
  ssh-keygen -t ed25519 -C "m1max"

  # Copy your public key to the Mac Mini
  ssh-copy-id y.song@10.11.33.81

  That's it. After entering your password one last time for ssh-copy-id, all future connections will be passwordless:

  ssh y.song@10.11.33.81   # No password prompt anymore

  If ssh-copy-id isn't available or fails, do it manually:

  cat ~/.ssh/id_ed25519.pub | ssh y.song@10.11.33.81 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

  You can also add a shortcut to ~/.ssh/config on your M1 Max so you just type ssh macmini:

  Host macmini
      HostName 10.11.33.81
      User y.song

  2. Remote Access from Outside School Network — Tailscale

  Yes, Tailscale is exactly the right tool for this. It creates a private encrypted tunnel between your devices using WireGuard, and it works through NATs and firewalls without opening any ports or doing anything the school network would flag.

  Why it won't trigger school security alerts:
  - Tailscale uses standard outbound HTTPS (port 443) to coordinate — looks like normal web traffic
  - The actual data tunnel uses WireGuard over UDP, which is standard VPN traffic
  - You're not opening inbound ports, running servers, or doing port forwarding
  - No changes to the school network infrastructure
  - It's the same technology many universities officially use

  Install Tailscale on Both Machines

  On your M1 Max:
  brew install tailscale
  Or download from https://tailscale.com/download/mac

  On the Mac Mini (SSH in first):
  ssh y.song@10.11.33.81
  brew install tailscale
  Or download from https://tailscale.com/download/mac

  Set Up Tailscale

  On both machines, open the Tailscale app (it installs as a menu bar app on macOS). Click "Log in" and sign in with the same account (Google, GitHub, or Apple
  ID — whatever is easiest).

  After both machines are logged in, each gets a stable Tailscale IP (100.x.x.x) that never changes and works from anywhere.

  Check your Tailscale IPs:
  # On either machine
  tailscale ip -4

  Say the Mac Mini gets 100.64.0.2. Now from anywhere in the world:

  ssh y.song@100.64.0.2   # Works from home, coffee shop, anywhere

  Update your ~/.ssh/config on the M1 Max to use the Tailscale IP:

  Host macmini
      HostName 100.64.0.2
      User y.song

  Keep Tailscale Running on the Mac Mini

  Since the Mac Mini needs to be reachable even when you're not sitting at it:

  1. Open System Settings > General > Login Items on the Mac Mini
  2. Make sure Tailscale is in the "Open at Login" list
  3. In the Tailscale menu bar icon, there's no special setting needed — it stays connected by default

  One Important Thing

  The Mac Mini needs to be awake to accept SSH connections. Prevent it from sleeping:

  # On the Mac Mini — disable sleep
  sudo pmset -a sleep 0 displaysleep 10 disksleep 0

  This keeps the Mac Mini awake 24/7 (screen turns off after 10 minutes to save energy, but the machine stays on).
